Introducción a la Programación

La programación es el proceso de escribir instrucciones que permiten a las computadoras realizar tareas específicas. A través del código, se crean aplicaciones, se automatizan procesos y se resuelven problemas de manera eficiente. Es un campo bastante amplio, con diferentes enfoques y lenguajes, cada uno diseñado para abordar distintas necesidades y tipos de proyectos. Al aprender a programar, puedes desarrollar soluciones tecnológicas que transforman ideas en herramientas útiles, facilitando tareas y mejorando la productividad en muchos ámbitos.

Este espacio está diseñado para ofrecerte una visión básica de la programación, cubriendo los fundamentos esenciales de forma simple y concisa. A lo largo de los contenidos, verás una breve explicación de los temas más importantes y tendrás acceso a tutoriales prácticos que te ayudarán a poner en práctica lo aprendido. Si eres nuevo en el mundo de la programación o quieres refrescar conceptos básicos, aquí encontrarás el punto de partida ideal.

¿Qué es la Programación Estructurada?

La programación estructurada es un estilo de programación que organiza el flujo de ejecución del programa de manera lógica, permitiendo que sea más comprensible y fácil de mantener. A diferencia de otros paradigmas como la programación orientada a objetos, se enfoca en el uso de estructuras de control y la organización secuencial del código

Principales características:

Secuencia: El código se ejecuta en orden, línea por línea, sin saltos inesperados.
Selección: El flujo del programa puede tomar diferentes caminos dependiendo de las condiciones (estructura if).
Iteración: Permite repetir acciones utilizando bucles (for, while).

Variables y Tipos de datos

Las variables son contenedores que almacenan datos y que pueden cambiar durante la ejecución del programa. Para declarar una variable, es necesario especificar el tipo de dato (números, caracteres, booleanos, etc.) que almacenará y un nombre único para la variable.

Tipos de Datos:

Entero (int): Almacena números enteros.
Flotante (float): Almacena números con decimales.
Doble (double): Similar a float, pero con mayor precisión.
Carácter (char): Almacena un solo carácter.
Booleano (bool): Solo puede ser true o false.

Ejemplo: 

int edad = 21; // Variable de tipo entero
float altura = 1.55; // Variable de tipo flotante
char text = 'Hola mundo'; // Variable de tipo carácter
bool esEstudiante = true; // Variable de tipo booleano

Estructuras de Control

Las estructuras de control son fundamentales en la programación estructurada. Permiten decidir qué instrucciones se deben ejecutar en función de ciertas condiciones o repetir bloques de código. Se dividen en estructuras condicionales y estructuras iterativas (bucles).

Condicionales (if, else, switch):
Las estructuras condicionales permiten ejecutar diferentes bloques de código según se cumpla o no una condición.

• if - else: Se usa para ejecutar un bloque de código si una condición es verdadera, y otro si es falsa.

int main() {
    int edad = 18;

    if (edad >= 18) {
        printf("Eres mayor de edad.\n");
    } else {
        printf("Eres menor de edad.\n");
    }

    return 0;
}

• if - else if - else: Se usa cuando hay múltiples condiciones a evaluar en cascada.

int main() {
    int nota = 75;

    if (nota >= 90) {
        printf("Tienes una A.\n");
    } else if (nota >= 80) {
        printf("Tienes una B.\n");
    } else if (nota >= 70) {
        printf("Tienes una C.\n");
    } else {
        printf("Reprobaste.\n");
    }

    return 0;
}

• switch - case: Se usa cuando hay varias opciones específicas para una variable.

int main() {
    int opcion = 2;

    switch (opcion) {
        case 1:
            printf("Seleccionaste opción 1.\n");
            break;
        case 2:
            printf("Seleccionaste opción 2.\n");
            break;
        case 3:
            printf("Seleccionaste opción 3.\n");
            break;
        default:
            printf("Opción no válida.\n");
    }

    return 0;
}

Bucles (for, while, do while):
Los bucles permiten ejecutar un bloque de código varias veces mientras se cumpla una condición.

• while: Ejecuta un bloque de código mientras la condición sea verdadera.

int main() {
    int i = 1;

    while (i <= 5) {
        printf("Iteración %d\n", i);
        i++;
    }

    return 0;
}

• do - while: Similar a while, pero siempre ejecuta el bloque al menos una vez.

int main() {
    int i = 1;

    do {
        printf("Iteración %d\n", i);
        i++;
    } while (i <= 5);

    return 0;
}

• for: Se usa cuando conocemos cuántas veces debe repetirse el bloque de código.

int main() {
    for (int i = 1; i <= 5; i++) {
        printf("Iteración %d\n", i);
    }

    return 0;
}

Funciones y Modularidad

Las funciones permiten dividir un programa en bloques de código más pequeños y reutilizables, facilitando la comprensión y el mantenimiento del código.

Definición de Funciones: Una función es un bloque de código que realiza una tarea específica. Puede recibir parámetros y devolver un valor.

int suma(int a, int b) {
    return a + b;
}
int main() {
    int resultado = suma(5, 3);
    printf("El resultado de la suma es: %d", resultado);
    return 0;
}

Funciones con parámetros y retorno: Las funciones pueden tomar uno o más parámetros y devolver un valor de cualquier tipo.

float areaCirculo(float radio) {
    return 3.1416 * radio * radio;
}
int main() {
    float area = areaCirculo(5);
    printf("El área del círculo es: %.2f", area);
    return 0;
}

Ventajas de la modularización:

Reutilización: El mismo bloque de código puede ser utilizado en distintas partes del programa.

Facilidad de mantenimiento: Si hay un error, solo es necesario modificar el código de la función, no en todos los lugares donde se usa.

Resumen y Conclusión

La programación estructurada ayuda a mantener el código limpio, modular y fácil de entender. Con el uso de variables, estructuras de control y funciones, los programadores pueden crear aplicaciones eficientes y fáciles de depurar.

Variables: Son esenciales para almacenar datos.
Estructuras de control: Permiten modificar el flujo de ejecución según condiciones.
Funciones: Facilitan la organización del código y permiten reutilizar tareas comunes.

La programación estructurada es una base esencial para cualquier programador, y aunque existen paradigmas más complejos como la programación orientada a objetos, su aprendizaje es fundamental para entender la lógica de programación.

Ejercicio 1: Positivo, Negativo o Cero

Este ejercicio permite al usuario ingresar un número y recibir una respuesta sobre si es positivo, negativo o cero.

Código:

// Función que verifica si el número es positivo, negativo o cero
function verificarNumero() {
    // Obtiene el valor ingresado en el campo de texto y lo convierte en número
    let num = parseFloat(document.getElementById("numeroInput").value);
    let resultado = ""; // Variable donde guardaremos el resultado de la verificación

    // Condicional para verificar si el número es positivo
    if (num > 0) {
        resultado = "El número es positivo.";
    } 
    // Condicional para verificar si el número es negativo
    else if (num < 0) {
        resultado = "El número es negativo.";
    } 
    // Si el número no es ni positivo ni negativo, es cero
    else {
        resultado = "El número es cero.";
    }

    // Muestra el resultado en el párrafo con el id "resultado1"
    document.getElementById("resultado1").innerText = resultado;
}

Ejercicio 2: FizzBuzz

Este ejercicio imprime los números del 1 al 100 y reemplaza ciertos números con las palabras "Fizz", "Buzz" o "FizzBuzz".

"Fizz": Si el número es divisible por 3.
"Buzz": Si el número es divisible por 5.
"FizzBuzz": Si el número es divisible por 3 y 5.
Si el número no es divisible ni por 3 ni por 5, simplemente muestra el número.

Código:

// Función que ejecuta el juego FizzBuzz
function mostrarFizzBuzz() {
    let resultado = ""; // Variable para almacenar los resultados

    // Bucle que recorre los números del 1 al 100
    for (let i = 1; i <= 100; i++) {
        // Si el número es divisible por 3 y 5, agregar "FizzBuzz"
        if (i % 3 === 0 && i % 5 === 0) {
            resultado += "FizzBuzz, ";
        } 
        // Si el número es divisible solo por 3, agregar "Fizz"
        else if (i % 3 === 0) {
            resultado += "Fizz, ";
        } 
        // Si el número es divisible solo por 5, agregar "Buzz"
        else if (i % 5 === 0) {
            resultado += "Buzz, ";
        } 
        // Si el número no es divisible ni por 3 ni por 5, agregar el número mismo
        else {
            resultado += i + ", ";
        }
    }

    // Muestra el resultado final en el párrafo con el id "resultado3"
    document.getElementById("resultado3").innerText = resultado.slice(0, -2);
}

¿Qué es la POO?

La Programación Orientada a Objetos (POO) es un paradigma de programación basado en la creación y manipulación de objetos. En lugar de estructurar el código en funciones y variables globales, en la POO se encapsulan datos y comportamientos dentro de clases y objetos.

Principios clave de la POO:

Facilita la reutilización del código.
Mejora la organización y el mantenimiento del software.
Representa objetos del mundo real de forma más intuitiva.

Clases y Objetos

Clases:

Son modelos o plantillas a partir de las cuales se crean los objetos.

Objetos:

Son instancias concretas de una clase, con sus propios valores y comportamientos.

Ejemplo:

// Definición de una clase "Persona"
class Persona {
    constructor(nombre, edad) {
        this.nombre = nombre; // Atributo
        this.edad = edad; // Atributo
    }

    // Método para mostrar información
    presentarse() {
        return `Hola, soy ${this.nombre} y tengo ${this.edad} años.`;
    }
}

// Creación de un objeto
let persona1 = new Persona("Carlos", 25);
console.log(persona1.presentarse()); // "Hola, soy Carlos y tengo 25 años."
×

Atributos y Métodos

Atributos:

Son las características o propiedades de un objeto.

Métodos:

Son las funciones que definen el comportamiento del objeto.

Ejemplo:

// Definición de la clase "Coche"
class Coche {
    constructor(marca, modelo) {
        this.marca = marca; // Atributo: Marca del coche
        this.modelo = modelo; // Atributo: Modelo del coche
    }

    // Método que indica que el coche ha sido encendido
    arrancar() {
        return `${this.marca} ${this.modelo} está encendido.`;
    }
}

// Creación de un objeto de la clase Coche
let miCoche = new Coche("Toyota", "Corolla");

// Llamar al método "arrancar" y mostrar el resultado
console.log(miCoche.arrancar()); // "Toyota Corolla está encendido."

Encapsulamiento: Protección de Datos

El encapsulamiento es uno de los pilares de la Programación Orientada a Objetos (POO). Su objetivo es proteger los datos de una clase y controlar cómo pueden ser accedidos o modificados desde fuera. Para lograr esto, los lenguajes de programación utilizan modificadores de acceso, que determinan la visibilidad de los atributos y métodos de una clase.

Modificadores de Acceso en POO

Público (public): Se puede acceder desde cualquier parte del programa.

Protegido (protected): Se puede acceder dentro de la clase y sus subclases.

Privado (private): Solo es accesible dentro de la misma clase.

Ejemplo:

// Definición de la clase "CuentaBancaria" con un atributo privado
class CuentaBancaria {
    #saldo; // Atributo privado: saldo de la cuenta

    constructor(saldoInicial) {
        this.#saldo = saldoInicial; // Inicializa el saldo con el valor dado
    }

    // Método para depositar dinero en la cuenta
    depositar(cantidad) {
        this.#saldo += cantidad; // Aumenta el saldo con la cantidad depositada
        return `Has depositado ${cantidad}. Saldo actual: ${this.#saldo}`;
    }

    // Método para retirar dinero de la cuenta
    retirar(cantidad) {
        if (cantidad > this.#saldo) {
            return "Saldo insuficiente."; // Verifica si hay suficiente saldo
        }
        this.#saldo -= cantidad; // Disminuye el saldo con la cantidad retirada
        return `Has retirado ${cantidad}. Saldo actual: ${this.#saldo}`;
    }

    // Método para obtener el saldo actual de la cuenta
    obtenerSaldo() {
        return `Tu saldo actual es: ${this.#saldo}`;
    }
}

// Creación de un objeto "cuenta" con un saldo inicial de 500
let cuenta = new CuentaBancaria(500);

// Realizar operaciones y mostrar los resultados
console.log(cuenta.depositar(200)); // "Has depositado 200. Saldo actual: 700"
console.log(cuenta.retirar(100));   // "Has retirado 100. Saldo actual: 600"

Herencia: Reutilización de Código

La herencia permite que una clase (llamada clase hija o subclase) adquiera los atributos y métodos de otra clase (llamada clase padre o superclase). Esto evita la repetición de código y facilita la extensión de funcionalidades.

Ejemplo:

// Definición de la clase padre "Animal"
class Animal {
    constructor(nombre) {
        this.nombre = nombre; // Atributo: nombre del animal
    }

    // Método que devuelve un sonido genérico de un animal
    hacerSonido() {
        return "Hace un sonido";
    }
}

// Definición de la clase hija "Perro", que hereda de "Animal"
class Perro extends Animal {
    // Sobrescribimos el método "hacerSonido"
    hacerSonido() {
        return "Ladra";
    }
}

// Creación de un objeto de la clase "Perro"
let miPerro = new Perro("Max");

// Llamar al método "hacerSonido" y mostrar el resultado
console.log(miPerro.hacerSonido()); // "Ladra"

Polimorfismo: Métodos con Distintos Comportamientos

El polimorfismo permite que un mismo método tenga diferentes comportamientos según la clase que lo implemente. Hay dos tipos principales:

Polimorfismo de Sobreescritura (Override): Cuando una subclase redefine un método de la clase padre.

Polimorfismo de Sobrecarga (Overload): Cuando varios métodos tienen el mismo nombre pero diferente cantidad o tipo de parámetros.

Ejemplo:

// Definición de la clase "Ave"
class Ave {
    // Método que indica que algunas aves pueden volar
    volar() {
        return "Algunas aves pueden volar.";
    }
}

// Definición de la clase "Aguila", que hereda de "Ave"
class Aguila extends Ave {
    // Se sobrescribe el método "volar"
    volar() {
        return "El águila vuela alto.";
    }
}

// Definición de la clase "Pinguino", que también hereda de "Ave"
class Pinguino extends Ave {
    // Se sobrescribe el método "volar" porque los pingüinos no vuelan
    volar() {
        return "El pingüino no puede volar.";
    }
}

// Creación de objetos de las clases "Aguila" y "Pinguino"
let aguila = new Aguila();
let pinguino = new Pinguino();

// Llamar al método "volar" y mostrar el resultado
console.log(aguila.volar());  // "El águila vuela alto."
console.log(pinguino.volar()); // "El pingüino no puede volar."

Resumen y Conclusión

La Programación Orientada a Objetos (POO) organiza el código mediante clases y objetos, promoviendo la reutilización y el mantenimiento eficiente del software. Los conceptos clave de POO incluyen:

Clases y Objetos: Las clases definen atributos y métodos, mientras que los objetos son instancias de esas clases.

Encapsulamiento: Protege los datos usando modificadores de acceso como public, private, y protected.

Herencia: Permite que una subclase herede atributos y métodos de una superclase, promoviendo la reutilización de código.

Polimorfismo: Un mismo método puede tener diferentes comportamientos dependiendo del objeto que lo invoque, mediante sobrescritura o sobrecarga.

La POO facilita la creación de software modular, escalable y fácil de mantener. Sus conceptos como herencia, polimorfismo y encapsulamiento son fundamentales para escribir código limpio y eficiente, utilizado en lenguajes como Java, C#, Python y JavaScript. Aprender POO es esencial para cualquier programador.

Ejercicio 1: Calcular el Área de un Círculo

Este ejercicio te enseñará a crear una clase con un atributo y un método que realice un cálculo (en este caso, el área de un círculo).

Código:

// Clase Círculo
class Circulo {
    constructor(radio) {
        this.radio = radio; // Atributo de la clase
    }

    calcularArea() {
        return Math.PI * Math.pow(this.radio, 2); // Fórmula para calcular el área
    }
}

// Función para calcular el área
function calcularArea() {
    const radio = parseFloat(document.getElementById("radio").value);
    const circulo = new Circulo(radio); // Crear un objeto de la clase Circulo
    const area = circulo.calcularArea();
    document.getElementById("resultado").textContent = `El área del círculo es: ${area.toFixed(2)} unidades cuadradas.`;
}

Ejercicio 2: Información de un Empleado

Este ejercicio te permitirá practicar el uso de clases con atributos más complejos y métodos que devuelvan información.



Código:

// Clase Empleado
class Empleado {
    constructor(nombre, salario) {
        this.nombre = nombre; // Nombre del empleado
        this.salario = salario; // Salario del empleado
    }

    mostrarInformacion() {
        return `${this.nombre} gana un salario de $${this.salario.toFixed(2)} al mes.`;
    }
}

// Función para mostrar la información del empleado
function mostrarEmpleado() {
    const nombre = document.getElementById("nombre").value;
    const salario = parseFloat(document.getElementById("salario").value);
    const empleado = new Empleado(nombre, salario); // Crear un objeto de la clase Empleado
    document.getElementById("resultadoEmpleado").textContent = empleado.mostrarInformacion();
}
